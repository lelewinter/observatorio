# LeetCode Resolve Apenas Metade das Entrevistas de Engenharia — System Design e Behavioral São a Outra Metade

## Resumo
Resolver 200-300 problemas de LeetCode é condição necessária mas não suficiente para passar em entrevistas de empresas de tecnologia de nível FAANG. A outra metade exige domínio de system design, behavioral questions e comunicação de trade-offs.

## Explicação
O problema com o foco exclusivo em LeetCode é que ele treina algoritmos, mas entrevistas sênior dedicam 50% ou mais do tempo a perguntas que algoritmos não respondem:

- **System Design**: como você projetaria o Twitter? O WhatsApp? O YouTube? Exige conhecimento de distributed systems, caching, filas, bancos de dados
- **Behavioral (STAR)**: como você lidou com conflito no time? Com uma feature que falhou em produção?
- **Trade-off communication**: conseguir explicar *por que* escolheu uma abordagem, não só *que* abordagem escolheu

**Analogia:** LeetCode treina o teste de dirigir em baliza. A entrevista real é uma estrada com trânsito, chuva e passageiros no carro.

O post lista ~28 recursos (via links encurtados) cobrindo: system design patterns, behavioral prep, comunicação técnica e guias de entrevista.

## Exemplos
- "Design a URL shortener" — questão típica de system design onde LeetCode não ajuda em nada
- "Tell me about a time you disagreed with your manager" — behavioral que requer framework STAR bem praticado
