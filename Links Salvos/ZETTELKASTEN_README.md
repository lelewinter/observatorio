
## Princípios Aplicados

### 1. Atomicidade
Cada nota contém **UMA ideia principal**. Notas não têm listas longas — em vez disso, cada item da lista vira uma nota separada.

✓ BOM: "Plan mode previne execução prematura"
✗ RUIM: "Claude Code - Melhores Práticas" (contém 6 ideias)

### 2. Títulos Específicos
Títulos são **afirmações ou perguntas específicas**, não genéricas.

✓ BOM: "Mistral lançou modelo TTS open-source que supera ElevenLabs"
✗ RUIM: "Mistral TTS - Text-to-Speech Local Gratuito"

### 3. Auto-Contenção
Cada nota pode ser **compreendida independentemente** sem ler outras. O frontmatter fornece contexto original.

### 4. Interconexão
Todas as notas relacionadas são **linkadas com [[]]**, criando uma rede navegável.

## Como Contribuir

Para adicionar novas notas:

1. **Respeite atomicidade**: Uma ideia por nota
2. **Use títulos específicos**: Não genéricos
3. **Siga o formato**: Frontmatter + título + conteúdo + links
4. **Sem listas longas**: Use prosa ou separe em notas

## Perguntas Frequentes

### Por que dividir notas?

Porque notas maiores com múltiplas ideias são:
- Mais difíceis de reutilizar
- Menos flexíveis para conexões
- Mais pesadas para revisar
- Menos efetivas para aprendizado

### E se eu quiser ler tudo sobre Claude Code?

Use a seção "Ver também" de qualquer nota de Claude Code. Os links o levarão através de todas as notas relacionadas, criando um caminho de aprendizado natural.

### Como buscar notas?

1. **Por tag**: Procure tags no frontmatter (ex: `[claude-code, ia]`)
2. **Por palavra-chave**: Ctrl+F no seu editor
3. **Por contexto**: Siga os links [[ ]]
