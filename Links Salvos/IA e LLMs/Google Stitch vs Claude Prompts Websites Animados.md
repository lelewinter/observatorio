# Google Stitch vs Claude: Comparação de Prompts para Websites Animados

## Resumo

Comparação de Viktor Oddy entre Google Stitch e Claude na geração de landing pages premium. Ambos produziram resultados visualmente atraentes e profissionais com um mesmo prompt para criar página SaaS com design "liquid glass". É como comparar dois restaurantes diferentes — ambos conseguem fazer pizza boa com mesma receita, mostrando que a receita (prompt) importa mais que o cozinheiro (modelo).

## Explicação

O teste utilizou um prompt para criar uma página landing dark e premium para empresa fictícia "APEX" — plataforma de aceleração de receita, usando design "liquid glass". Google Stitch produziu versão visual polida com design moderno, animações suaves e estética premium. Claude também produziu resultado polido com design competitivo, qualidade comparable e animações similares.

**Analogia:** Antes, cada modelo era como diferente ferro de passar roupa — uns queimavam, uns amassavam, você tinha que aprender cada um. Agora: prompt estruturado é como roupa bem feita — qualquer ferro de marca decente consegue passar bem. O que importa é mais a roupa (prompt) que o ferro (modelo).

O post ilustra competição acirrada entre ferramentas, demonstrando que Google Stitch e Claude produzem qualidade similar. Ambos conseguem criar landing pages premium, mesmos prompts podem ser usados em diferentes modelos, e ferramentas específicas como MotionSites surgem para aproveitar capacidades de geração.

**Profundidade:** Por que qualidade é similar? Porque ambos os modelos (Google Stitch, Claude) atingiram patamar de competência similar em web generation. Diferença agora não é "qual modelo é melhor", é "qual é mais rápido, qual custa menos, qual é mais acessível". Isso muda mercado: prompts bons valem mais que modelos bons, porque prompts funcionam em múltiplos modelos.

Ambos geram layouts estruturados, paletas de cores dark premium, animações "liquid glass" e elementos de SaaS (metrics, benefits, CTAs). O visual sugere que o mercado de geração de website com IA está maduro, com especialização em prompts por caso de uso, bibliotecas de templates, otimização de prompts para diferentes modelos e automação completa de landing pages.

## Exemplos

MotionSites.ai (http://motionsites.ai) oferece acesso a todos os prompts para websites animados em um clique, com biblioteca de prompts prontos para landing pages animadas e ferramenta centralizada para criar landing pages.

Exemplo de caso de uso: para quem trabalha com design de landing pages, SaaS marketing, web design acelerado, prototipagem visual. Ter acesso a prompts exatos que funcionam bem em múltiplos modelos permite escolher melhor ferramenta por caso, reutilizar prompts eficazes, acelerar ciclo de design e reduzir iterações de refinement.

Prompt: criar página landing dark e premium para empresa fictícia chamada "APEX" — plataforma de aceleração de receita, usando design "liquid glass".
